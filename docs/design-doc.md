## 1. Overview

### 目的
PdMとしての経験とAI活用を通じて、地域課題の解決アプローチを発信する。
トモさん自身の実践を見せることで、読者が「自分でもできる」と思って動き出すきっかけをつくる。

### メディア名
**地域課題、どう解く？**
*まちに、プロダクト思考を。*

### ドメイン（仮）
`localpdm.jp` または `local-pdm.com`（正式決定前の仮）

### 一言コンセプト
> 地域課題は、プロダクト思考で解ける。

### ターゲット
地域活動に関わりたい人（関係人口）
- 地域おこしに興味があるが、どう関わればいいか分からない人
- 地域課題に対してテクノロジーやプロダクト思考を活かしたい人
- 喜茂別・北海道の地域活動に関心を持つ人

## 2. コンテンツ設計

### セクション構成

| セクション | 概要 | 更新頻度 |
|---|---|---|
| 地域課題の構造と分析 | PdM視点で地域課題を言語化・構造化する記事 | ロングテール狙い・頻度より質 |
| プロトタイプのつくりかた | 地域課題を事例にしたプロダクト思考の実践ガイド。課題発見→仮説→プロト→学びのサイクルを実名フィールドで解説 | ロングテール狙い・頻度より質 |
| 実際にやってみた | 作ったもの・試したことのレポート。AI活用実践含む | ロングテール狙い・頻度より質 |
| 活動ログ | 喜茂別での動きなど。等身大の記録 | 不定期 |
| このメディアについて | 発信者・目的・メディア自体の成り立ち |  |

### 記事の種類
- **分析記事** — 課題をフレームワークで構造化
- **プロトレポート** — 作ったものの記録（課題→仮説→結果→学び）
- **AI活用レポート** — 実際に試したツール・プロンプト・ワークフロー
- **活動ログ** — 現地での動き・気づき

### コンテンツ方針
- かっこつけない。等身大で書く
- 失敗や未完成も記録する
- 「メディアを作る過程」自体もコンテンツにする
- **思考はトモさん、執筆はAI** — Claudeを使った記事化ワークフロー自体も発信対象
- **更新頻度は気にしない** — ロングテール狙いで良い記事を積み上げる。定期更新より質と蓄積を優先

### コンテンツループ

このメディアのコンテンツは以下のループで設計する。
プロダクト開発のサイクルそのものが、地域課題解決の方法論であることを構造で示す。

```
事実（経験・直面したこと）
　↓
分析（地域課題の構造と分析）
　↓
仮説・プロトタイプ（プロトタイプのつくりかた）
　↓
検証（実際にやってみた）
　↓
また新しい事実へ…
```

**ゴミ問題を例にすると：**
- 事実 → ゴミ拾いしてもまた捨てられる
- 分析 → なぜ捨てられるのか？行動経済学・環境設計の視点で構造化
- 仮説・プロト → 地域通貨、看板デザイン、トイレ設置など
- 検証 → 実際に試してみた結果と学び

### 記事文字数の目安

| セクション | 目安文字数 |
|---|---|
| 地域課題の構造と分析 | 3,000〜5,000文字 |
| プロトタイプのつくりかた | 3,000〜5,000文字 |
| 実際にやってみた | 2,000〜4,000文字 |
| 活動ログ | 1,000〜2,000文字 |

### 執筆ワークフロー

```
活動・会議・現地メモ（ソースは何でもいい）
　↓
Claudeで構造化・記事化
　↓
Markdown → GitHub管理
　↓
Next.jsで表示（MDX）
```

## 3. ターゲットユーザー

### ペルソナ
- 都市部在住、30〜40代
- 地域に関わりたいが接点がない
- テクノロジーやビジネス的な視点を持っている
- 「移住」ではなく「関わる」という選択肢を探している

### ユーザーのジョブ
- 地域課題の解像度を上げたい
- 自分が地域でできることのヒントを得たい
- 同じような視点を持つ人とつながりたい

## 4. 実験フィールド

このメディアの記事は、以下の実際の活動・場を実験フィールドとして書かれる。
抽象的な地域論ではなく、**実名・実践ベース**であることがこのメディアの強み。

| フィールド | 課題領域 |
|---|---|
| Tigris（カフェ・ゲストハウス） | 地域拠点運営・関係人口・観光 |
| 青年部 | コミュニティ運営・合意形成 |
| 学校教育 | 教育×地域・次世代へのアプローチ |
| 子育て | 地方での子育て環境・コミュニティ |

## 5. 技術スタック

- **フレームワーク:** Next.js (App Router)
- **UI:** shadcn/ui + Tailwind CSS
- **コンテンツ管理:** Markdown + GitHub（MDX）。記事はClaudeが生成、GitHubで管理
- **ソース（エビデンス）:** Notion議事録・現地メモ・会話・写真など何でも可。特定ツールに依存しない
- **ホスティング:** AWS Amplify（学習コストと運用コストのバランスが最適）
- **その他:** 必要に応じてCMS検討
