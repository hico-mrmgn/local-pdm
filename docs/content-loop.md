# コンテンツループ

このメディアのコンテンツは「事実 → 分析 → 仮説・プロト → 検証」のループで設計する。
プロダクト開発のサイクルそのものが、地域課題解決の方法論であることを構造で示す。

```
事実（経験・直面したこと）
　↓
分析（地域課題の構造と分析）
　↓
仮説・プロトタイプ（プロトタイプのつくりかた）
　↓
検証（実際にやってみた）
　↓
また新しい事実へ…
```

## ループとセクションの対応

| ステージ | セクション | 例（ゴミ問題） |
|---|---|---|
| 事実 | 活動ログ / 実際にやってみた | ゴミ拾いしてもまた捨てられる |
| 分析 | 地域課題の構造と分析 | 環境設計・心理・インセンティブの3層で構造化 |
| 仮説・プロト | プロトタイプのつくりかた | 地域通貨、看板、トイレ設置 |
| 検証 | 実際にやってみた | 実装結果と学び |

## 記事を書くときの問い
- これは事実か、分析か、仮説か、検証か？
- 一つ前のステージの記事と紐づくか？
- 次のステージで書く余地を残せているか？

ループを意識すると、記事同士が自然と相互リンクし、メディア全体の蓄積になる。
