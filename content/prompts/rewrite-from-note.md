# プロンプト: 過去の note 記事をリライトする

過去に書いた note の記事を、このメディアの文体・コンテンツループに沿った形でリライトするためのプロンプト。

## このリライトで大事にしたいこと

単に文体を整えるリライトではない。**当時の自分（書いた時点での思考）と、いまの自分（リライト時点の理解）の二層を、同じ記事のなかで対比させる**。

そうすることで、

- 書き手にとっては「あの時はこう考えていたのか」という思考の整理になる
- 読者にとっては「人の理解は時間とともに変わる」というプロセスそのものが価値になる

ので、リライト記事には **当時の解釈** と **いまの解釈** を両方とも明示することを必須とする。

---

## ユーザープロンプト（テンプレート）

```
以下は過去に note で書いた記事です。これを「地域課題、どう解く？」用にリライトしてください。

【セクション】
{analysis | prototype | experiments | logs}

【コンテンツループ上の位置づけ】
事実 / 分析 / 仮説・プロト / 検証 のどれか

【リライトの方針】
- 文体方針（system prompt 参照）に従う
- 元記事の体験・事実は残す
- **当時の自分が何を考えていたかを、ぼかさず明示する**（元記事の言葉・解釈をそのまま拾うか、当時の感覚を一段落で言語化する）
- **いまの解釈を、当時の解釈と対比する形で書く**（「当時はこう考えていた → いまはこう見える」の差分を明確にする）
- プロダクト思考の構造化を加える（症状と課題を分ける、フレームワークで分解する など）
- 「次のステップ」をコンテンツループの次ステージに接続する一段落を入れる

【元記事 URL】
{URL}

【元記事本文】
（必要なら貼る）

【当時の思考メモ】
（リライト時点で覚えている「当時こう考えていた」を1〜3行で。AIに想像で書かせない）

---

出力フォーマット:
- 冒頭に MDX frontmatter
- 本文は Markdown（h2 区切り）
- 構成の目安:
  1. 体験・事実から入る
  2. 「当時はこう考えていた」を明示するセクション or 段落
  3. 「いま見直すと」の構造化・分析
  4. 差分から見える学び
  5. 次のステップ（コンテンツループの次ステージへ）
```
